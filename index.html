Conteúdo

Vamos falar sobre alguns comandos Git, tanto na teoria como na prática. Seguem os comandos:

Comando init

    Cria um repositório git vazio na pasta, junto de um branch sem commits. É essencial para o início da funcionaliade do Git.

    Uso: git init

    Exemplo Prático:

Comando clone

    Obtem um repositório a partir de um link e adiciona a um diretório novo. É usado para pegar repositórios de lugares como GitHub.

    Uso: git clone [seu link]

    Exemplo Prático:

Comando branch

    O comando branch possui diversos usos dependendo dos seus parâmetros.

    Comando Descrição
    git branch Lista todos os branches do repositório atual.
    git branch [seu branch] Cria um novo branch com o nome digitado.
    git branch -d [seu branch]    Deleta o branch com o nome digitado.

    Exemplo Prático:

Comando checkout

    Move entre diferentes branches.

    Uso: git checkout [seu branch]

    Exemplo Prático:

Comando status

    Lista todos arquivos que ainda devem ser "commitados"

    Uso: git status

    Exemplo Prático:

Comando diff

    Mostra as diferenças do arquivo em relação ao index (área de estágio para o próximo commit).

    Uso: git diff

    Exemplo Prático:

Comando add

    Adiciona um arquivo para a área de estágio.

    Uso: git add [nome do arquivo]

    Exemplo Prático:

Comando commit

    Usado para o controle de versões, o commit guarda os arquivos encontrados área de estágio no histórico de versões.

    Uso: git commit -m [mensagem/nome para o commit]

    Também pode ser usado o modificador -a para incluir todos arquivos modificados além dos adicionados com o git add.

    Exemplo Prático:

Comando push

    Envia seu commit para uma branch de um repositório remoto como o GitHub.

    Uso: git push [seu repositório remoto] [qual branch]

    Exemplo Prático:

Comando pull

    Busca mudanças no repositório remoto e combina estas mudanças ao seu diretório de trabalho.

    Uso: git pull [seu repositório remoto] [qual branch]

    Exemplo Prático:

Comando revert

    Desfaz algum commit específico sem remover outros commits posteriores. O resultado é um novo commit.

    Uso: git revert [commit a ser desfeito]

    Exemplo Prático:

Comando merge

    Combina o histórico de um branch ao branch atual.

    Uso: git merge [nome do branch]

    Exemplo Prático:

Comando stash

    O comando stash possui diversos usos dependendo dos seus parâmetros.

    Comando Descrição
    git stash save Guarda todos arquivos modificados temporáriamente.
    git stash pop Restaura os arquivos guradados.
    git stash list Lista todas mudanças guardadas.
    git stash drop    Descarta as mudanças guardadas.

    Exemplo Prático:

Comando rm

    Deleta um arquivo do seu diretório de trabalho e registra esta remoção no estágio.

    Uso: git rm [seu arquivo]

    Exemplo Prático:

Comando config

    É usado para configurar que nome e email serão usados em seus commits.

    Comando Descrição
    git config –global user.name [seu nome] Configura seu nome.
    git config –global user.email [seu email]    Configura seu email.

    Exemplo Prático:

Comando reset

    O comando reset possui diversos usos dependendo dos seus parâmetros.

    Comando Descrição
    git reset [nome do arquivo] Tira um arquivo da área de estágio sem mexer no seu conteúdo.
    git reset [nome do commit] Desfaz todos commits depois do commit digitado e guarda-os localmente.
    git reset -hard [nome do commit]    Desfaz todos commits depois do commit digitado, sem guarda-los.

    Exemplo Prático:

Comando remote

    Para evitar ter que adicionar um link toda vez, o remote permite definir uma variável como seu repositório remoto.

    Uso: git remote add [nome da variável] [link do repositório]
